Major Steps in Exploratory Data Analysis (EDA)

  1. Data Loading and Inspection:

    • Loaded the dataset and inspected its structure, including column names, data types, and initial rows.

  2. Data Cleaning:

    • Identified and handled missing values.
    • Removed duplicate entries and irrelevant columns.

  3. Exploratory Visualizations:

    • Created histograms, box plots, and scatter plots to understand data distributions and relationships.
    • Visualized correlations between features using heatmaps.

  4. Feature Engineering:

    • Generated new features based on domain knowledge.
    • Encoded categorical variables and normalized numerical features.

  5. Outlier Detection:

    • Identified and addressed outliers using statistical methods and visualizations.

  6. Text Handling:

    • Preprocessed text data by cleaning, tokenizing, and removing stop words.
    • Extracted numerical features from text using techniques like TF-IDF and BERTopic.
    • Analyzed word frequencies and created topic models for better insights.

  7. SHAP Analysis:

    • Computed SHAP values to interpret model predictions and understand feature contributions.
    • Visualized SHAP results using summary plots, force plots, and dependence plots.
    • Identified key features driving predictions and validated their importance.

Handling Missing Data Approach

  1. Leverage XGBoosts native handling For most features, I allow XGBoost to natively handle missing values, since its tree-splitting mechanism can route missing values optimally without explicit imputation. This prevents unnecessary bias from manual filling.

  2. Diagnosis columns with extreme sparsity (>95% missing) Instead of keeping hundreds of ultra-sparse diag_* columns, I collapse them into broader binary flags (e.g., any diagnosis in last 2 years). This reduces dimensionality and transforms rare but clinically meaningful information into model-usable features.

  3. Measurement columns with heavy missingness (>50%) If a measurement (lab or vital) is missing in more than half of the patients, I drop it entirely.

  4. Measurement columns with moderate missingness (<50%) In an ideal settingand especially without domain expertsthe best practice is to use single or multiple imputation methods (e.g., ML-based iterative imputation) that model relationships among variables. However, due to time constraints, I use median imputation for these columns. Median is robust to outliers, easy to reproduce, and avoids leakage when fit on the training split only.

Here's a concise summary of the modeling pipeline implemented in my topicModel function, explaining each component's role, rationale for its inclusion, and the assumptions behind your choices:

1. Sentence Embedding Model

Model: SentenceTransformer(embedding_model_name) Chosen model: "paraphrase-multilingual-MiniLM-L12-v2" Purpose: Converts each Hebrew clinical note (text) into a numerical vector embedding that captures semantic meaning.

Why this model:

  • Supports Hebrew and other multilingual content.
  • Lightweight and efficient for clinical notes.

Assumption: Sentence embeddings capture clinical similarity even across variations in phrasing.

2. CountVectorizer

Model: CountVectorizer(...) Purpose: Creates bag-of-words representations (with n-grams) for calculating class-based TF-IDF per topic.

Settings:

  • ngram_range=(1, 2) captures unigrams and bigrams (e.g., )
  • stop_words=he_clinical_stop removes Hebrew clinical stopwords
  • min_df=1, max_df=0.6 filters out extremely rare or overly common terms

Assumption: Term frequency patterns help identify distinctive words per topic.

3. ClassTfidfTransformer (c-TF-IDF)

Model: ClassTfidfTransformer(...) Purpose: Weights words per topic, instead of per document. Helps in identifying representative terms for each cluster.

Why used: c-TF-IDF is better suited than standard TF-IDF when dealing with topic-level aggregation.

Assumption: Distinctive terms can be inferred by comparing word usage across topics.

4. UMAP (Dimensionality Reduction)

Model: umap.UMAP(...) Purpose: Reduces the high-dimensional embedding vectors into a lower-dimensional space to help with clustering and visualization.

Why UMAP:

  • Preserves both global and local structure
  • Faster and more scalable than t-SNE

Assumption: Topics form meaningful clusters in a reduced latent space (e.g., 5D with n_components=5).

5. HDBSCAN (Clustering)

Model: hdbscan.HDBSCAN(...) Purpose: Finds natural groupings (topics) in the reduced embedding space.

Settings:

  • min_cluster_size=30 ensures clusters are large enough to be meaningful
  • metric='euclidean' aligns with UMAP output

Why HDBSCAN:

  • Handles noise (i.e., irrelevant or vague topics)
  • Doesnt force every point into a cluster

Assumption: Clinical documents naturally group into semantic clusters (e.g., gestational hypertension, diabetes, etc.).

6. Maximal Marginal Relevance (MMR)

Model: MaximalMarginalRelevance(diversity=0.7) Purpose: Selects diverse yet relevant keywords to describe each topic.

Why: Prevents repetitive or overly similar keywords for a single topic.

Assumption: High diversity leads to better interpretability of topics.

7. BERTopic

Model: BERTopic(...) Purpose: Integrates all components (embedding, dimensionality reduction, clustering, vectorizer, and TF-IDF) to produce:

  • Document-level topic assignments
  • Top keywords per topic
  • Probability scores (if enabled)

Why BERTopic:

  • Modular, interpretable topic modeling
  • Supports multilingual data (including Hebrew)
  • Scalable to real-world EHR data

Assumptions:

  • Latent topics exist in the data and can be captured meaningfully using unsupervised learning.
  • Each document can belong to a single dominant topic.

def normalize_phrases(text: str) -> str:
    text = "" if text is None else str(text)
    phrase_map = {
        r"\b \b": "_",
        r"\b (|)\b": "_",
        r"\b  \b": "__",
        r"\b \b": "_",
        r"\b \b": "_",
        r"\b \b": "_",
        r"\b \b": "_",
    }
    for pat, repl in phrase_map.items():
        text = re.sub(pat, repl, text)
    return text

def clean_clinical_hebrew(s: str) -> str:
    s = normalize_phrases("" if s is None else str(s))
    s = re.sub(r"^\s*[:\-]?\s*", " ", s, flags=re.IGNORECASE | re.MULTILINE)
    s = re.sub(r"^\s*[:\-]?\s*", " ", s, flags=re.IGNORECASE | re.MULTILINE)
    s = re.sub(r"\s+\d+\s+", " ", s)
    s = re.sub(r"(?:)?\s+\d+", " ", s)
    s = re.sub(r"\b\d+[.,]?\d*\b", " ", s)
    s = re.sub(r"\b(dL|mg|mmHg|kg|cm)\b", " ", s)
    s = re.sub(r"\s{2,}", " ", s).strip()
    return s
